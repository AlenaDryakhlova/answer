Руководство пользователя 
Программное обеспечение: Android Studio  
(Соответствует ГОСТ Р 59795–2021)  
1. Введение
1.1 Область применения
Android Studio — интегрированная среда разработки (IDE) для создания мобильных приложений под ОС Android. Используется разработчиками для написания, тестирования и отладки кода.  
1.2 Краткое описание возможностей
- Поддержка языков программирования: Java, Kotlin, C++.  
- Встроенный эмулятор Android.  
- Инструменты для отладки и профилирования.  
- Готовые шаблоны проектов.  
- Интеграция с Firebase и Google Cloud.  
1.3 Уровень подготовки пользователя 
Пользователь должен обладать базовыми знаниями в программировании и работе с ОС Windows/Linux/macOS.  
1.4 Перечень документации
- Официальная документация Android Developers.  
- Руководство по установке и настройке. 
2. Назначение и условия применения
2.1 Виды деятельности
- Разработка мобильных приложений.  
- Тестирование и отладка кода.  
- Прототипирование интерфейсов.  
2.2 Условия применения
- ОС: Windows 10/11, macOS, Linux.  
- Технические требования:
  - 8 ГБ ОЗУ (рекомендуется 16 ГБ).  
  - 4 ГБ свободного места на SSD.  
  - Поддержка виртуализации (для эмулятора).  
3. Подготовка к работе
3.1 Установка
1. Скачать установщик с [официального сайта](https://developer.android.com/studio).  
2. Запустить файл `android-studio-<версия>.exe`.  
3. Следовать инструкциям мастера установки.  
3.2 Первый запуск 
1. Выбрать настройки (стандартные или кастомные).  
2. Установить необходимые SDK и эмулятор.  
3. Проверить работоспособность через создание тестового проекта. 
4. Описание операций 
4.1 Создание нового проекта
1. Файл → New → New Project.  
2. Выбрать шаблон (например, Empty Activity).  
3. Указать имя проекта, язык (Kotlin/Java) и минимальную версию SDK.  
4. Нажать Finish.  
4.2 Запуск эмулятора 
1. Tools → Device Manager → Create Device. 
2. Выбрать модель телефона и версию Android.  
3. Запустить эмулятор кнопкой ▶.  
5. Аварийные ситуации 
5.1 Ошибки при запуске 
- Проблема: Эмулятор не запускается.  
- Решение: Проверить включен ли Hyper-V/Virtualization в BIOS.  
5.2 Потеря данных
- Проблема: Проект не сохраняется.  
- Решение: Использовать File → Save All и систему контроля версий (Git).  
6. Рекомендации по освоению
- Изучите официальные туториалы Google.  
- Используйте Logcat для отладки.  
- Тестируйте приложение на реальных устройствах.
